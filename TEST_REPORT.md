## Выявленные замечания

### 1. Курс Car Repair (ID 72) пустой
**Статус:** Не критично, UX улучшен в ходе тестирования  
**Описание:** Переход на `/courses/72` открывает страницу курса без модулей. API `core_course_get_contents` возвращает пустой массив для данного курса.  
**Действие:** В `CourseDetailView.vue` добавлено сообщение для пользователя:
> "В этом курсе пока нет материалов."

**Рекомендация:** Проверить, засеян ли курс 72 в Moodle (возможно, требуется запуск `seed` для этого курса).

### 2. Автотест: селекторы кнопки "Начать тест"
**Статус:** Исправлено в автотесте v3  
**Описание:** Стандартный `page.click()` Playwright не срабатывал на кнопке "Начать тест" в компоненте `QuizPlayer.vue`. Причина — особенность рендеринга Vue/Shadow DOM или перекрытие элементов.  
**Действие:** В автотест добавлен JS-fallback (`btn.click()` через `page.evaluate()`), после чего тест надежно запускается.

### 3. Автотест: навигация по многостраничному тесту
**Статус:** Исправлено в автотесте v3  
**Описание:** Первые версии скрипта выбирали ответ только на 1-м вопросе и сразу искали кнопку "Завершить тест", которой на тот момент не было (отображалась "Далее").  
**Действие:** Добавлен цикл автоматического ответа на все вопросы с кликом "Далее" до появления кнопки "Завершить тест".

### 4. Moodle WS `core_user_get_users` — 500
**Статус:** Известная проблема, не блокирующая  
**Описание:** Интермитентный HTTP 500 при поиске пользователя админ-токеном. Не влияет на прохождение тестов студентом.

Добавлен плейсхолдер для пустого курса:

<p v-if="sections.length === 0 || sections.every(s => !(s.modules && s.modules.length))" class="empty-course">
  В этом курсе пока нет материалов.
</p>
