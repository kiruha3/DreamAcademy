# Правила репозитория DreamDocs Academy

## Что НИКОГДА не коммитить

### 1. Секреты и ключи
- `.env` файлы (включая `.env.local`, `.env.production`)
- API токены Moodle (`MOODLE_TOKEN`)
- Пароли баз данных, SSH-ключи, сертификаты
- Файлы `*.pem`, `*.key`, `*.p12`, `*.pfx`

### 2. Локальные данные и volumes
- Данные Docker volumes (`db_data/`, `moodle_data/`, `moodledata_data/`)
- Загруженный пользовательский контент (`uploads/`)
- Локальные логи (`logs/`, `*.log`)

### 3. Зависимости и артефакты сборки
- `node_modules/`
- `__pycache__/`, `*.pyc`, `*.egg-info/`
- `dist/`, `build/`, `target/`

### 4. Автогенерируемые отчёты
- `graphify-out/` (пересобирается локально)
- Папки покрытия тестов (`htmlcov/`, `.coverage`)

## Процедура перед коммитом

1. **Проверить `.gitignore`**: убедиться, что новые секреты/артефакты в нём
2. **Запустить тесты**: `pytest tests/backend/ -v`
3. **Code review**: прочитать diff перед `git push`
4. **Никаких `.env` в коммите**: только `.env.example` с плейсхолдерами

## Контакты при инциденте

Если секрет случайно попал в git — немедленно ротировать токен/пароль и переписать историю (`git filter-repo` или BFG).
