## Баг #1: Студент не видит содержимое курса Car Repair (ID 72)

**Суть:**
- Админ открывает `/courses/72` и видит полноценный курс "Базовый ремонт автомобиля" с 23 модулями.
- Студент (тест v3, аккаунт `kiruha3@mail.ru`) открывает тот же URL и получает пустую страницу с сообщением "В этом курсе пока нет материалов."

**Причина:**
API `core_course_get_contents` возвращает пустой массив для пользователя, который не заэнроллен в курс. Админ имеет глобальный доступ, поэтому Moodle отдаёт ему содержимое без enrolment.

**Влияние:** Студенты не могут пройти обучение по курсу Car Repair.

**Статус:** Требует фикса.

## Баг #2: URL `/admin/courses` не существует

**Суть:**
В роутере Vue (`router/index.js`) прописан только путь `/admin`. Переход на `/admin/courses` (логичный для админа) приводит к пустому layout: шапка + футер, но без контента.

**Причина:**
Отсутствует маршрут `/admin/courses` или редирект на `/admin`.

**Влияние:** Путаница администратора, создаётся впечатление, что админка сломана.

**Статус:** Уже пофикшено (добавлен редирект).

## План фикса

### 1. Автоматическая запись (enrolment) на курс Car Repair (72)
**Приоритет:** High  
**Варианты решения:**

- **A. Авто-энролл при регистрации**
  - В `register_router.py` (или где создаётся пользователь) после создания `moodle_user_id` вызвать `enrol_user(72, moodle_user_id)`.
  - *Плюс:* прозрачно для студента.
  - *Минус:* нужно знать список "обязательных" курсов.

- **B. Кнопка "Записаться на курс" в UI**
  - На странице пустого курса (или в каталоге) показывать кнопку записи, если студент не заэнроллен.
  - Бэкенд: эндпоинт `POST /api/courses/{id}/enrol`.
  - *Плюс:* универсальное решение для любых курсов.
  - *Минус:* требует доработки и фронта, и бэка.

- **C. Авто-энролл при первом открытии курса**
  - В `course_content.py` → `get_course_content` перед выдачей контента проверять enrolment и автоматически записывать пользователя через `moodle_db` или Moodle WS.
  - *Плюс:* минимум изменений в UI.
  - *Минус:* side-effect на GET-запросе.

**Рекомендуемый фикс:** Комбинация **B + C**:
1. Добавить эндпоинт `POST /api/courses/{course_id}/enrol`.
2. На фронте в `CourseDetailView.vue` при получении пустого курса показывать кнопку "Записаться на курс".
3. После успешной записи перезагружать содержимое курса.

### 2. Редирект `/admin/courses` → `/admin`
**Приоритет:** Low  
**Статус:** ✅ Уже применён.

В `portal/frontend/src/router/index.js` добавлена строка:
```js
{ path: '/admin/courses', redirect: '/admin' },
```

Теперь переход по `/admin/courses` корректно ведёт на админ-панель.

### 3. Backend эндпоинт для записи на курс
**Приоритет:** High  
**Статус:** ✅ Уже применён.

В `portal/backend/app/modules_router.py` добавлен эндпоинт:
```python
@router.post("/{course_id}/enrol")
async def enrol_in_course(...)
```

Использует `MoodleClient.enrol_user()` с `role_id=5` (студент).

### 4. UI-кнопка "Записаться на курс"
**Приоритет:** High  
**Статус:** ✅ Уже применён.

В `portal/frontend/src/views/CourseDetailView.vue` добавлена кнопка "Записаться на курс", которая отображается, когда курс пустой (нет модулей). После успешной записи страница перезагружается и загружает актуальное содержимое.

**Примечание:** бэкенд-контейнер перезагружается автоматически (uvicorn `--reload`), а фронтенд требует перезапуска dev-сервера (`npm run dev`) для применения изменений Vue-роутера и компонентов.

### 3. Проверка `get_interactive_elements()` в SkreenMaker
**Приоритет:** Low (тестовый инструмент)  
**Суть:** В некоторых сценариях `BrowserController.get_interactive_elements()` падает с `'list' object has no attribute 'tag'`.  
**Действие:** В автотесте v4 добавлен `try/except` вокруг вызова — скриншоты делаются в raw-режиме.
